⑤ Image audit — portfolio use cases
Thumbnail Use case card thumbnails — left panel of work cards
  • ✓Background must use Warm Canvas #F7F6F2 or Linen #EAE9E4 — never white #FFFFFF or arbitrary off-whites. Screenshots on pure white will look cold and disconnected from the site.
  • ✓Content area padding: 24px uniform inset on all sides (--space-6). The mockup or graphic must breathe inside the frame.
  • !Border radius on the thumbnail container must match card's --radius-lg (12px). Any screenshot placed as a flat rectangle inside a rounded card will look jarring.
  • !No drop shadows on elements inside the thumbnail — the card already uses --shadow-card. Adding inner shadows doubles the depth and fights the minimal aesthetic.
  • ✓Accent colors in thumbnail graphics must be drawn from the palette — Teal #2BA5A5, Rose #D4797A, or Deep Tide #1C2B3A only. Any third-party screenshot should be color-adjusted or placed as-is without added decoration in off-brand colors.
  • →Recommended thumbnail aspect ratio: 4:3 (matches the ~340×255px slot visible in the work cards). Do not use 16:9 — it will require letterboxing or cropping that wastes the space.
Inline Case study body images — screenshots, flows, annotations
  • !Every screenshot placed inside a case study must be wrapped in a browser/device frame or a neutral card container — bare screenshots with visible OS chrome look unfinished. Use a simple border-radius: 12px; border: 0.5px solid rgba(28,43,58,0.08) container on Warm Canvas.
  • ✓Max content width: 900px (--container-max). Full-bleed images wider than this should be centered with the same max-width or cropped to focus on the relevant UI.
  • !Annotation styles must be consistent throughout the case study. Use only Teal #2BA5A5 for callout lines, highlights, or numbered markers. Never red, orange, or any off-brand color for annotations.
  • !Caption text must use Jost 300, 9px, Slate #6B7A8D (--text-caption). Do not use system default caption styling from Figma or Notion exports — it will conflict.
  • ✓For UI flow diagrams and wireframes: background must be Warm Canvas or Linen. Dark backgrounds are only acceptable when the product itself is dark-themed.
  • →Images should be exported at 2× resolution (retina-ready). At the 900px max-width, export at 1800px wide, then compress with WebP at 85% quality.
What's working From the portfolio screenshots shared
  • ✓The existing thumbnail for "Building a public presence" correctly uses a light background with simplified UI mockup — good precedent to follow for all thumbnails.
  • ✓The locked case study uses Deep Tide #1C2B3A as its thumbnail background with centered iconography — consistent with the dark surface token.
  • ✓Typography on the homepage is correctly pairing Cormorant Garamond italic for the hero with Jost 300 for body copy — images placed in the case study should not break this rhythm.
Common fixes needed Issues to address before publishing images
  • !Logo SVG file is empty — the uploaded file contains only a blank 344×207 white rectangle with no paths. Please re-export from Figma as SVG with "Include ids" off and verify the file contains path data before use.
  • !Color contamination risk: Any screenshot taken from a product with its own color scheme (e.g. Avast's brand colors, NetSuite's UI) should be placed as-is without additional colored frames. Adding a teal or rose border to a screenshot that already has competing brand colors creates visual noise.
  • !Consistent framing: All thumbnails must use the same frame style — either all use a flat card container or all use a device mockup. Mixing both within the same "Work" grid makes the page feel inconsistent.
  • !Spacing inside thumbnails: Leave at least 16px (--space-4) between any content element and the thumbnail edge. Elements that bleed to the edge read as cropped, not designed.

⑦ Hard rules for every image
Never use pure white backgrounds
Use #F7F6F2 or #EAE9E4. Pure white #FFFFFF breaks the warm canvas tone of the site.
No off-brand accent colors
Annotations, highlights, and UI elements in images must use #2BA5A5, #D4797A, or #1C2B3A only.
Slate is light-surface only
#6B7A8D must never appear on Deep Tide backgrounds. Use #A8B5C2 for dark surfaces instead.
Rose is never used as text
#D4797A is decorative and interactive only — buttons, accents, the logo dot. Never for body or caption text in images.
Font consistency in graphics
Any text visible in images (labels, annotations, legends) must use Jost at 300 or 400 weight only.
One shadow level only
Never nest shadows. If the card has a shadow, elements inside it must not also have shadows. Use the --shadow-card token only.
